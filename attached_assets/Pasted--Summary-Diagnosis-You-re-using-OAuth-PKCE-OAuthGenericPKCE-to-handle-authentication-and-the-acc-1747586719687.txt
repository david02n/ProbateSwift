✅ Summary Diagnosis
You’re using OAuth PKCE (OAuthGenericPKCE) to handle authentication, and the access token is required to be attached to requests via a bearer header (in BearerAuthedRequest).

In production, it seems:

The OAuth login completes, but

The token is either not set correctly or not being passed with the request to /api/user

Therefore, the backend (e.g. Express) returns a 401 Unauthorized

3. ⚠️ The problem is likely this:
Your this.token is not set in ChromeNetwork at the time /api/user is called — either:

The token is not stored correctly after OAuth

Or the request is made before the token is set
