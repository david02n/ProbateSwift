2. Back-End API Updates
Endpoints & Models

Rename any references of executors → people in routes, controllers, services, and DB models.

Update request/response schemas to use first_names, last_name, address_line1, postcode, executor, etc.

CRUD operations

GET /cases/:id/people (was /executors): return all people for a case.

POST /cases/:id/people: create a new person entry.

PUT /cases/:id/people/:personId: update fields, including partial saves (needs_more_info).

DELETE /cases/:id/people/:personId: remove a person.

Self-suggestion flag

If you have an endpoint for case settings, add support for storing dismissSelfSuggestion: boolean.
